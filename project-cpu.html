Digital Design · Embedded Systems

8-bit CPU

ABEL FSM Digital Logic Embedded Systems

Overview

Designed and implemented a fully functional 8-bit CPU from scratch. The architecture was specified and implemented in ABEL (Advanced Boolean Expression Language), covering all major subsystems of a classic processor: arithmetic/logic, memory addressing, program flow, and sequenced control. Each component was designed, verified independently, then integrated into a working processor capable of executing a custom instruction set.

Arithmetic Logic Unit (ALU)

The ALU handles all computation on 8-bit operands. The supported operations include addition, subtraction, bitwise AND/OR, and logical shifts. Output flags — zero, carry, and overflow — are generated combinationally and fed to the control unit to support conditional branching. Careful attention was paid to two's complement overflow detection and correct carry propagation across the full 8-bit width.

Program Counter & Program Memory Address Unit (PC/PMAU)

The PC tracks the address of the currently executing instruction and is incremented each cycle in normal execution. The PMAU computes the effective program memory address, supporting both sequential fetch and branch targets. On a taken branch, the PC is loaded with the branch target address rather than PC+1, enabling conditional and unconditional jumps within the program.

Data Memory Address Unit (DMAU)

The DMAU manages address generation for load and store instructions. It computes the effective data memory address from instruction fields and register values, then presents this to the data memory bus. Separating the DMAU from the PMAU ensures that instruction fetch and data memory access can be sequenced without conflict under FSM control.

Control Unit & Finite State Machine (FSM)

The control unit is the central sequencer of the CPU, implemented as a finite state machine in ABEL. It steps through fetch, decode, and execute stages for each instruction, asserting the correct control signals — register enables, ALU operation select, memory read/write, PC load — at each cycle. The FSM was designed to handle all instruction types cleanly, including multi-cycle operations like load/store and conditional branches that require different sequencing paths depending on ALU flag state.

Instruction Set

The CPU supports a minimal but complete instruction set:

  • Arithmetic: ADD, SUB with flag generation
  • Logic: AND, OR, shift left/right
  • Memory: LOAD (memory → register), STORE (register → memory)
  • Control flow: unconditional JUMP, conditional branch on zero/carry/overflow flags

Design Challenges

The most demanding aspect was ensuring the FSM correctly sequenced all instruction types without introducing hazards. Particular care was needed for:

  • Coordinating the PMAU and DMAU so instruction fetch and data access don't overlap during load/store sequences.
  • Ensuring ALU flags were latched at the correct cycle so conditional branches read stable values.
  • Verifying that the reset state of the FSM brought the CPU to a known, clean starting condition regardless of initial register contents.
