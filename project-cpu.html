Digital Design · EE

8-bit CPU

ABEL FSM Digital Logic Embedded Systems

Overview

Designed and implemented a fully functional 8-bit CPU from the ground up. The architecture was specified and implemented in ABEL (Advanced Boolean Expression Language), covering all major components of a classic processor pipeline — from arithmetic to memory addressing to control flow.

Components

  • ALU (Arithmetic Logic Unit) — handles all arithmetic and logical operations on 8-bit operands, including addition, subtraction, AND, OR, and shift operations.
  • PC / PMAU (Program Counter / Program Memory Address Unit) — tracks the current instruction address and computes the next address, supporting sequential execution and branching.
  • DMAU (Data Memory Address Unit) — manages addressing into data memory for load and store operations.
  • Control Unit with FSM — a finite state machine that sequences through fetch, decode, and execute stages, generating the appropriate control signals for each component at each step.

Design Approach

Each component was designed and verified independently before integration. The FSM-based control unit was central to the architecture — it had to correctly sequence control signals across all instruction types while handling edge cases like branching and memory wait states.

The instruction set was kept minimal but complete: arithmetic, logic, memory access (load/store), and conditional branching were all supported. ABEL allowed specifying the logic equations and state transitions directly, which made it straightforward to reason about timing and correctness.

Key Challenges

  • Ensuring the FSM correctly handled all instruction types without hazards or incorrect sequencing.
  • Coordinating the PMAU and DMAU so that instruction fetch and data access didn't conflict.
  • Verifying ALU flag behavior (zero, carry, overflow) and propagating them correctly into conditional branch logic.
